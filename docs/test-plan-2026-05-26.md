## Post-Test Verification

After all 8 sessions, verify training data was collected:

```bash
# Check S5 decision count
wc -l ~/.cynco/training/s5-decisions.jsonl

# Check governance DB
ls -la ~/.cynco/governance/governance.db

# Check prediction data
sqlite3 ~/.cynco/governance/governance.db "SELECT hypothesis, COUNT(*) as n, SUM(CASE WHEN correct=1 THEN 1 ELSE 0 END) as correct FROM predictions GROUP BY hypothesis"

# Check session outcomes
sqlite3 ~/.cynco/governance/governance.db "SELECT outcome, COUNT(*) FROM sessions GROUP BY outcome"
```

**Expected minimums after all 8 sessions:**
- S5 decisions: 200+ entries
- Governance DB: 8 session records
- Predictions: Some H1/H2/H6 triggers (variety/stuck/homeostat are most common)
- Session outcomes: Mix of viable/marginal

## Training Data Quality Checklist

For each session, the S5 decision journal should contain:
- [ ] Input context (user message, workflow, context %, governance state)
- [ ] Decision output (tools, contextAction, priority, reasoning, ruleIds)
- [ ] Outcome backfill (did the decision lead to good results?)

For prediction tracking:
- [ ] At least some H1 (variety) predictions triggered
- [ ] Predictions have both trigger and evaluation data
- [ ] `/governance report` shows non-zero sample counts

For the fine-tune pipeline:
- [ ] `~/.cynco/training/s5-decisions.jsonl` has (input, decision, outcome) triples
- [ ] Rule weights updated in `~/.cynco/training/s5-weights.json`
- [ ] GovernanceDB has session-level outcome data
